# 湘超地图项目互动页面功能清单及重构分析

## 1. 现有互动页面功能模块梳理

通过对 GitHub 上 `paofuxiaomiao/xiangchao-map` 项目源码（主要涉及 `FeatureModules.tsx` 和 `H5Experience.tsx`）的分析，目前该项目的“互动中心”（或 H5 体验页）主要包含以下核心功能模块：

### 1.1 球队与数据展示（数字看板 Dashboard）
该模块作为信息展示的基础，提供了球队及赛事的全面数据。
- **球队总览**：展示球队排名、积分、主教练信息、阵容规模（包含平均身高、体重）、得分焦点以及主场标语和文化主线。
- **阵容档案**：详细列出主教练风格及每位球员的具体信息（球衣号、位置、身高、体重、进球数），并附带球员当前的综合评分。
- **联赛榜单**：展示联赛各项数据维度的领先者（Leaders）以及包含所有球队热度（投票数）与积分的综合排行榜。

### 1.2 互动与评价中心（Interactive）
这是用户参与度最高的模块，涵盖了多种互动形式，主要基于本地存储（localStorage）实现状态留存。
- **球队热度投票**：用户可为支持的球队投票，系统会显示实时（或本地模拟的）各球队总票数及排名。目前逻辑限制每台设备只能投出一票。
- **线上打气（应援）**：提供预设的打气模板（如特定口号），用户点击即可增加对应口号的热度（Heat），模拟现场看台的应援氛围。
- **球员打分系统**：用户可以为球队的每位球员进行 1-5 分的评价，系统结合基础分和用户打分计算出球员的平均得分。
- **比赛评论与动态**：
  - **留言板**：用户可输入昵称、选择球队并发表赛前打气或赛后评价，留言会按时间倒序展示。
  - **实时战报（Live Reports）**：穿插展示正在进行或已结束比赛的简报，包含比分、比赛时间状态（如 LIVE、PREVIEW、FULL TIME）及赛况总结。

### 1.3 主场文化展示（Culture）
侧重于球队所在城市的文化背景及看台组织介绍。
- **城市文化锚点**：结合特定城市（如株洲的炎帝文化、永州的舜帝文化）展示定制化背景及文化说明。
- **看台组织与仪式**：介绍球队的啦啦队、粉丝队职责，展示主场核心仪式及代表性横幅（Banner Samples），增强球迷的文化认同感。

## 2. 当前交互设计存在的问题分析

根据对代码结构和现有模块的观察，目前的交互设计在以下几个方面存在明显的“混乱”或优化空间：

1. **信息层级与导航割裂**：
   - 现有的 `FeatureModules` 采用了类似政务后台的 Tab 切换（Dashboard, Interactive, Culture），这对于一个面向球迷的互动应用来说显得过于生硬。
   - 用户在查看球队数据时，无法自然地过渡到为该球队打气或评论，必须在顶部视图间频繁切换。

2. **互动操作缺乏连贯性**：
   - 投票、打分、留言和打气虽然同属互动范畴，但在页面上被简单地堆砌成多个面板（PortalPanel）。
   - 例如，用户在“阵容档案”中看到球员表现后，需要切换到“互动中心”才能进行打分，操作路径不够直觉。

3. **视觉焦点分散**：
   - 在 H5 体验版中采用了长页面的滚动叙事（Snap Scrolling），但在全功能模块页中则变成了网格状的卡片平铺，导致用户在不同设备上的体验不一致，且重点信息（如焦点战报、热门投票）容易被淹没在大量数据中。

## 3. 下一步重构建议

为了提升互动页面的用户体验，建议从以下几个方向进行交互重构：

- **以“比赛/球队”为核心的沉浸式体验**：打破原有的功能模块硬分类，将数据、文化和互动融合。例如，在展示某场焦点战或某支球队的主页时，直接在下方自然衔接投票、打气和留言功能。
- **优化互动反馈链路**：将打气（点赞）、投票等轻量级互动做成悬浮或吸底的快捷操作，让用户在浏览球队故事或实时战报时能随时参与。
- **重塑信息架构**：采用“主从结构”，左侧或上方展示核心视觉（如球队文化图腾、实时比分），右侧或下方作为动态互动流（信息流模式展示留言、战报和打分动态），使页面更具现代社交媒体的活力。
